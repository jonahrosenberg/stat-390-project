git checkout model.py

Editable (agent modifies)
Keep (improved)

2 Risk List

# Risk Severity Mitigation How to Detect
1 Agent modifies frozen files
Agent edits prepare.py or run.py, invalidating the evaluation		 High Explicit instructions in program.md; prepare.py and run.py are not listed as editable; can add file permission locks git diff --name-only after each iteration — only model.py should appear
2 Model overfits / explodes
High-degree polynomials or unconstrained models produce wild predictions (e.g. RMSE = 5.0)		 High Keep/discard logic automatically reverts bad models; regularization (Ridge) as default RMSE jumps > 2x baseline → automatic discard
3 Metric is noisy / non-reproducible
Random seed not fixed → same model gives different RMSE on different runs		 Medium RANDOM_SEED = 42 frozen in prepare.py; data split is deterministic; all models use random_state=42 Run the same model.py twice — RMSE should be identical
4 Agent breaks the interface
Agent renames build_model() or changes its return type, causing import errors		 Medium program.md specifies: function signature must not change; run.py will crash with a clear error Runtime ImportError or AttributeError in run.py
5 Training exceeds time budget
Agent uses very large ensemble or expensive model (e.g. SVR on 16k samples)		 Low 60-second time budget stated in program.md; run.py prints training time for monitoring Training time > 60s in run.py output
6 Agent adds external dependencies
Agent imports packages not available in the environment (xgboost, lightgbm, etc.)		 Low program.md states: only use scikit-learn, numpy, matplotlib ImportError on python run.py

3 Project Repository Structure

demo-autoresearch/
├── prepare.py FROZEN — Data loading, train/val split, evaluation metric, plotting
├── model.py EDITABLE — Model pipeline definition. The only file the agent modifies.
├── run.py FROZEN — Experiment runner: train → evaluate → log to results.tsv
├── program.md FROZEN — Agent instructions: rules, constraints, search strategy
├── demo.py FROZEN — Pre-scripted 8-iteration simulation (for classroom demo)
├── README.md — Project documentation, setup, agent loop example
├── .gitignore — Excludes results.tsv, performance.png, __pycache__
├── results.tsv GENERATED — Experiment log (auto-appended by run.py, not committed)
└── performance.png GENERATED — RMSE/R² trajectory plot (generated by prepare.py)
Boundary principle: Everything the agent can change is in EDITABLE (only model.py). Everything that measures success is in FROZEN. This separation is what makes the research loop trustworthy.
